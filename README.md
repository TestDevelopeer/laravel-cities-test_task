Развертывание:

В папке docker_laravel_cities в файле docker-compose.yml выбрать БД (mysql \ pgsql) убрав комментирование и
закомментировав ненужное

В папке docker_laravel_cities изменить .env (опционально):

DB_DATABASE=db_cities

DB_USERNAME=user

DB_PASSWORD=password

NGINX_PORT=94

DB_PORT=3394 #(mysql) 5493 (pgsql) порты ставлю исходя из порта NGINX, просто для своего удобства

PROJECT_NAME=laravel-cities

После настройки .env запустить сборку:

cd ./docker_laravel_cities
docker-compose up -d --build

После сборки изменить подключение к БД в корневом файле laravel .env:

DB_CONNECTION=mysql

DB_HOST=host.docker.internal # ставим так, чтобы не приходилось выполнять миграции и прочую работу с бд через терминал
контейнера

DB_PORT=3394 # как в настройках докера ставили

DB_DATABASE=db_cities # как в настройках докера ставили

DB_USERNAME=user # как в настройках докера ставили

DB_PASSWORD=password # как в настройках докера ставили

Проект будет доступен по адресу http://localhost:ВАШ_ПОРТ_ИЗ_НАСТРОЙКИ_ДОКЕРА

Цель:
Создать модуль мультирегиональности с использованием городов. Необходимо реализовать маршрутизацию по городам, выделение
выбранного города и перенаправление на соответсвующий город, если он был выбран ранее.

Задачи:

1. Отображение списка городов на главной:
    - Список городов должен быть загружен из БД
    - Каждый город должен иметь уникальную ссылку:
        - Лондон: site.com/london
        - Париж: site.com/paris
    - При клике на город:
        - Открывается страница с тем же списком городов
        - Выбранной город выделяется жирным шрифтом
        - Если пользователь заходит на страницу по url без города и до этого уже выбирал город, сделать редирект на
          выбранный ранее город, пример site.com -> site.com/london
2. Запоминание города:
    - Если пользователь вводит URL с городом вручную (site.com/london), система должна автоматически запомнить этот
      город, при налчии его в БД, даже если ранее был уже выбран другой город
3. Страницы:
    - На каждой странице должен отображаться город в шапке.
    - Ссылки на страницы должны указываться без slug города:
        - Если выбран город london: маршрут route('news') должен вести на site.com/news
4. Не использовать slug напрямую в маршрутах:
    - Роуты должны быть объявлены без указания slug:
        - Route::get('/', [MainController::class, 'index'])->name('index');
        - Route::get('/about', [MainController::class, 'about'])->name('about');
        - Route::get('/news', [MainController::class, 'news'])->name('news');
    - При этом маршруты должны цеплять выбранный город в ссылках в меню шапки, показывая ссылку site.com/london/news

Структура страниц:

1. Главная:
    - Header: с отображением выбранного города
    - Content: список всех городов
2. О нас:
    - Header: с отображением выбранного города
    - Content: lorem
3. Новости:
    - Header: с отображением выбранного города
    - Content: lorem
